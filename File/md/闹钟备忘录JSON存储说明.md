# 闹钟和备忘录独立JSON文件存储说明

## 功能概述

闹钟和备忘录数据现在分别存储在独立的JSON文件中，便于管理和备份。

## 文件位置

- **闹钟数据**: `alarms.json`（与程序同目录）
- **备忘录数据**: `memos.json`（与程序同目录）

## 文件格式

### alarms.json
```json
[
  {
    "time": "08:00",
    "label": "起床闹钟",
    "enabled": true,
    "last_triggered": "2024-01-01 08:00"
  },
  {
    "time": "14:30",
    "label": "会议提醒",
    "enabled": false
  }
]
```

### memos.json
```json
[
  {
    "title": "购物清单",
    "content": "牛奶、面包、鸡蛋、水果..."
  },
  {
    "title": "工作任务",
    "content": "完成项目报告\n准备演示文稿\n回复客户邮件"
  }
]
```

## 新增功能

### 1. 刷新按钮
在闹钟和备忘录管理窗口底部添加了**🔄 刷新**按钮，点击后从JSON文件重新加载最新数据。

### 2. 自动保存
所有操作（添加、编辑、删除、切换状态）都会自动保存到对应的JSON文件。

### 3. 优化的列表显示

#### 闹钟列表
- **大字体标题**：显示闹钟标签（14号粗体）
- **小字体详情**：显示时间和启用状态（10号常规字体）
- **示例**：
  ```
  起床闹钟          （大字体）
  ⏰ 08:00 | 已启用  （小字体）
  ```

#### 备忘录列表
- **大字体标题**：显示备忘录标题（14号粗体）
- **小字体内容**：显示约3行内容（10号常规字体，自动换行）
- **示例**：
  ```
  购物清单                    （大字体）
  牛奶、面包、鸡蛋、水果...    （小字体，最多3行）
  ```

## 数据迁移

如果您之前使用settings.json存储闹钟和备忘录数据，可以手动迁移：

1. 打开 `settings.json`
2. 复制 `"alarms"` 数组到 `alarms.json`
3. 复制 `"memos"` 数组到 `memos.json`
4. 保存两个新文件

## 注意事项

1. **不要手动编辑JSON文件**，除非您了解格式要求
2. 如果JSON文件格式错误，程序会使用空列表作为默认值
3. 刷新按钮可以重新加载外部修改的JSON文件
4. 所有修改都会立即保存，无需手动保存操作

## 技术实现

- 使用Python内置的 `json` 模块进行读写
- UTF-8编码，支持中文
- 格式化输出（indent=2），便于阅读
- 异常处理，确保程序稳定性
